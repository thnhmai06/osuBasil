# BasilBot commands

Below are the chat commands of `BasilBot` — usable directly through chat (public channels, match channels, or DM to `BasilBot`).

Default prefix is `!` (server admin can change it via `BotOptions.CommandPrefix` in `settings.toml`). **Direct DM to BasilBot is an exception — prefix is optional there**: typing `help` or `!help` in DM are handled identically (every DM sent to BasilBot is already treated as a command; there is no fallback to regular chat/mail, so relaxing the prefix here carries no risk of swallowing normal messages).

For why specific differences from original osu! Bancho exist (deliberate vs. scope-related), see [`working-scopes.md`](working-scopes.md).

**Command listings in `!help`/`!mp help` are auto-generated by BasilBot from a single source in code** (not hand-written help strings) — adding a new command to that source auto-updates the help, no second location to edit. `!help` lists only general commands (+ `make`/`join`/`in`/`mp help` because these four work outside match scope); `!mp help` lists the match-scoped `!mp` subcommands separately. Some commands have optional numeric descriptions (e.g. `!mp set`, `!mp mods`, `!mp team`) — those lines wrap into separate messages when sent, like any multi-line reply.

## General commands

Usable anywhere (public channel, match channel, DM), no special permissions required.

| Command | Syntax | Description | Comparison with BanchoBot |
|---|---|---|---|
| Help | `!help` | Returns the list of general commands BasilBot supports (auto-generated from code, not privilege-dependent) | BanchoBot lists many more commands (`stats`, `report`, `request`...) that BasilBot does not have |
| Roll | `!roll [max]` | Returns a random number in `0..max` (inclusive). `max` defaults to `100` if omitted, max `2147483647`. Negative, `0`, or unparseable values fall back to `100` | BanchoBot randomizes `1..max` (different lower bound — never returns `0`). Same default `100` |
| Where | `!where <username>` | Shows the country of `username` (queried from the server's user database, works even when offline) | BanchoBot also shows city if the player has location sharing enabled — BasilBot shows country only |
| FAQ | `!faq <entry>` / `!faq list` | Prints the contents of a FAQ file prepared by the server admin (each line = separate message); `list` enumerates every available entry. Entry names may contain spaces; only `\` and `..` are blocked | BanchoBot supports 2-character language prefixes before entry names (e.g. `ru:lines`) for translations — BasilBot does not; only one version per entry |

```
!help
→ BasilBot: "!roll [max] - roll a random number from 0 to max (default 100)"
→ BasilBot: "!where <username> - show a player's country"
→ BasilBot: "!faq <entry>|list - print a FAQ entry, or list every entry"
→ BasilBot: "!mp make <name> - create a tournament room from anywhere, scoping you to it"
→ BasilBot: "!mp in [match_id] - target/show a match you're not physically in (needs referee rights there)"
→ BasilBot: "!mp help - list multiplayer subcommands (usable while scoped to a match)"
   (each line above is a separate message — the original reply is a multi-line string, each line split into its own message)

!roll
→ BasilBot: "PlayerOne rolls 42 point(s)"

!roll 50
→ BasilBot: "PlayerOne rolls 17 point(s)"

!where PlayerOne
→ BasilBot: "PlayerOne is in Vietnam"
   (if username does not exist → "GhostUser is not registered.")

!faq list
→ BasilBot: "Available FAQ entries: rules, schedule"
   (no entries yet → "No FAQ entries available.")

!faq rules
→ BasilBot sends each line of rules.txt as a separate message
   (entry not found → "No FAQ entry found for 'rules'.")
```

## Multiplayer commands

All commands for controlling matches — creating rooms, changing maps, managing players, starting matches...

### Referee and Host

**Referee** and **host** are two completely separate roles — neither automatically includes the other:

| | Referee | Host |
|---|---|---|
| Who are they? What can they do? | A person granted Referee status (`!mp addref`), authorized to use `!mp` commands on that match | A person physically present in the room, who can change settings directly in the client (not via chat) |
| Must be in the room? | **No** — a referee can control from a distance via `!mp in <match_id>` | **Yes** |
| Default assignee? | The room creator — whether via `!mp make` or the `Create Room` button in-game | The room creator (normal `Create Room` button) |

**The room creator is always both host and referee from the moment of creation**, regardless of whether the room was created via `!mp make` or the `Create Room` button in-game.

**Every `!mp` subcommand except `help`/`make`/`join`/`in` requires you to be a referee of the currently targeted match** — lacking permission means BasilBot **silently ignores the command with no reply** (easy to mistake for lag; it's actually missing permissions).

**Which match are you currently scoped to?** By default, the match linked to the chat channel you're in. To control a match without being in its room → use `!mp in <match_id>` (see section 1 below); once set, this scope **always takes priority** over the current channel until you change scope.

**Room lifecycle differs by creation method:**

- Rooms created via `!mp make` **do not auto-close when empty** — they only close via `!mp close`, or when the last referee is removed (`!mp removeref`).
- Rooms created normally in-game still auto-close when empty like original osu! Bancho, regardless of whether referees remain.

### Chaining multiple commands

Referees can chain multiple `!mp` subcommands in a single message (does not apply to `make`/`join`/`in`/`help`, nor to `!roll`/`!where`/`!faq`):

| Operator | Meaning |
|---|---|
| `;` | Next command always runs, regardless of whether the previous one succeeded |
| `&&` | Next command runs only if the immediately preceding command succeeded |

```
!mp lock; !mp start 30
→ BasilBot: "Locked the match"
→ BasilBot: "Match starts in 30 seconds"
   (each subcommand replies on its own line, joined by \n in the same message — both always execute,
    even if lock fails for any reason)

!mp map 123 && !mp start
→ countdown only starts if map change succeeds; if map fails, BasilBot only replies with the map error,
   !mp start does not run so there is no line for it
```

To use `;`/`&&` **literally** inside a parameter (not as command separator) → wrap in `"..."`:

```
!mp name "Round 1; Group A"
```

Within double-quoted strings, use `\"` for literal `"` and `\\` for literal `\`.

**If any segment of the chain is not a valid `!mp` subcommand** (e.g. mixed with `!roll`, or containing `!mp make`) BasilBot rejects the entire chain — no segment runs, preventing partial application that might mislead you into thinking everything was applied.

### 1. Create, join & target a match

| Command | Syntax | Description |
|---|---|---|
| Make | `!mp make <room name>` | Creates a new match. Can be used anywhere, including DM to BasilBot |
| Join | `!mp join <id> [password]` | Join a public match by its wire id (0-63). Private rooms cannot be joined this way — you must be invited. |
| In | `!mp in [match_id]` | Switches scope to another match without physically being in it |
| Private | `!mp private [0\|1]` | Show or set the room's private status. Private rooms are hidden from `#lobby` and require an invite to join. Referee only |

```
!mp make Group A: Team Alpha vs Team Beta
→ BasilBot: "Created the match #42 Group A: Team Alpha vs Team Beta. You are now scoped to this match, and
   added as a referee."
   (server full, cannot create → "Couldn't create the match — server is full.")
```

You are automatically placed in slot 0 as host **and** added as a referee. BasilBot replies with the **match ID** (`#<id>`) — remember this ID for `!mp in` later or for external API lookups (see [`api-external.md`](api-external.md)). No limit on simultaneous rooms. No name provided → defaults to `"<your name>'s match"`.

```
!mp in 42
→ BasilBot: "Now targeting match #42 Group A: Team Alpha vs Team Beta."
   (not a referee of match #42 → "You're not a referee of match #42.")
   (id does not exist → "No active match with id #42.")

!mp in
→ BasilBot: "Currently scoped to match #42 Group A: Team Alpha vs Team Beta."
   (never set scope → "You're not scoped to any match.")
```

```
!mp join 5
→ BasilBot: "Joined match #5 test match"
   (match not found → "No active match with id 5.")
   (room is private → "Cannot join match #5 — the room is private.")
   (already in a match → "You're already in a match.")
   (banned → "You're banned from this match.")

!mp join 5 secret123
→ BasilBot: "Joined match #5 test match"
   (wrong password → "Failed to join the match.")

!mp private
→ BasilBot: "This match is not private"
   (non-referee → silent no-op)

!mp private 1
→ BasilBot: "The match is now private. It will be hidden from the lobby."

!mp private 0
→ BasilBot: "The match is now public."
```

### 2. Room configuration

| Command | Syntax | Description |
|---|---|---|
| Settings | `!mp settings` | Shows room name, map, team type, win condition, mods, and slot list |
| Lock | `!mp lock` | Locks the room, prevents new players from joining |
| Unlock | `!mp unlock` | Unlocks the room |
| Private | `!mp private [0\|1]` | Show or set the room's private status. Private rooms are hidden from `#lobby` and can only be joined via invite |
| Size | `!mp size <1-16>` | Sets the number of available slots |
| Move | `!mp move <name> <slot 1-16>` | Moves a player to another slot |
| Host | `!mp host <name>` | Transfers host to another player |
| Clear host | `!mp clearhost` | Clears the current host |
| Name | `!mp name <new name>` | Renames the room (truncated at 50 characters) |
| Password | `!mp password [password]` | Sets a room password; omit to clear the password |
| Invite | `!mp invite <name>` | Sends an invite to an online player |
| Team | `!mp team <name> <red\|blue>` | Assigns a team to a player |
| Map | `!mp map <beatmap id>` | Changes the selected map, unreadies all players |
| Mods | `!mp mods <mods>` | Sets common mods for the entire match |
| Set | `!mp set <teammode 0-3> [scoremode 0-3] [size 1-16]` | Sets team mode + win condition + size in one command |

**`!mp settings`** — multi-line reply, one field per line:

```
!mp settings
→ BasilBot: "Room name: Group A: Team Alpha vs Team Beta (#42)"
→ BasilBot: "Beatmap: 111222 Camellia - Frontier [Insane]"
→ BasilBot: "Team mode: TeamVs, Win condition: ScoreV2"
→ BasilBot: "Active mods: DoubleTime, Freemod"     (only shown when mods/freemod are active)
→ BasilBot: "Players: 2"
→ BasilBot: "Slot 1  Not Ready 7 PlayerOne       [Host]"
→ BasilBot: "Slot 4  Not Ready 12 PlayerTwo       [NoFail]"
```

> **Note:** Original osu! Bancho attaches links to `osu.ppy.sh/mp/{id}` (match history) and player profiles to these lines — Basil does not have these two pages (offline, no public history/profile), so it shows raw text/numbers: `#42` instead of a history link, raw user IDs (`7`, `12`) instead of profile links. To look up a username from an ID, use `GET /users/{id}` from the admin API (see [`api-external.md`](api-external.md#42-user)) or the TRT snapshot (`GET /multi/{id}`, field `live.slots[].userId`). The trailing tag per slot (`[Host]`, `[NoFail]`, `[Host / NoFail]`) combines host role + personal mods under freemod.

```
!mp size 8
→ BasilBot: "Changed match to size 8"

!mp move PlayerOne 3
→ BasilBot: "Moved PlayerOne into slot 3"

!mp password
→ BasilBot: "Removed the match password"

!mp password abc123
→ BasilBot: "Changed the match password"
   (does not echo the password back)

!mp host PlayerTwo
→ BasilBot: "Changed match host to PlayerTwo"

!mp clearhost
→ BasilBot: "Cleared match host"

!mp name Semi-Finals
→ BasilBot: "Room name updated to \"Semi-Finals\""

!mp invite PlayerTwo
→ BasilBot: "Invited PlayerTwo to the room"
   (user not found online → "User not found: PlayerTwo")
   (user already in room → "User is already in the room")

!mp team PlayerOne red
→ BasilBot: "Moved PlayerOne to team Red"

!mp map 111222
→ BasilBot: "Changed beatmap to Camellia - Frontier"
   (id not in local DB → "No beatmap with id 111222 found locally.")
```

> **`!mp map`** only accepts `<beatmap id>` — original osu! Bancho also accepts a `<gamemode>` parameter to change the game mode simultaneously; BasilBot deliberately omits this parameter (see comparison table at end of section 2).

**`!mp mods`** — 3 usage modes:

```
!mp mods HR DT HD NF
→ BasilBot: "Enabled NoFail, Hidden, HardRock, DoubleTime, disabled FreeMod"
   (list sorted by bit-value ascending — .NET [Flags] enum default format, not your input order.
    Lists only mods that were NEWLY toggled, not all active mods — use !mp settings to see the full list)

!mp mods Freemod
→ BasilBot: "Enabled FreeMod"
   (enables freemod using existing mods: speed mods (DT/HT/NC) stay at match level, remaining mods
    become individual host mods. Adding extra mods in the same command, e.g. "!mp mods HD DT Freemod",
    is NOT applied — BasilBot only enables freemod, ignoring accompanying mod tokens in this case)

!mp mods None
→ BasilBot: "Disabled NoFail, Hidden, HardRock, DoubleTime, disabled FreeMod"
```

> Supported 2-character mod codes: `NF, EZ, HD, HR, SD, DT, RX, HT, NC, FL, SO, AP, PF` (see `!mp help` for BasilBot to list them). Unrecognized codes (e.g. `S2`, `FI`) are silently ignored.

**`!mp set`** — only `<teammode>` is required; `scoremode`/`size` are optional:

```
!mp set 2 3 16
→ BasilBot: "Changed match settings to 16 slots, TeamVs, ScoreV2"

!mp set 0
→ BasilBot: "Changed match settings to HeadToHead, Score"
   (only changes team mode; win condition keeps current value, not reset to "Score";
    no size arg → no "N slots," prefix in the response)
```

> Teammode: `0` HeadToHead, `1` TagCoop, `2` TeamVs, `3` TagTeamVs. Scoremode: `0` Score, `1` Accuracy, `2` Combo, `3` ScoreV2 — **no `pp` option**, server does not use pp as win condition.

> Compared to original osu! Bancho: all commands in section 2 behave identically, except `map` (missing `<gamemode>` parameter, deliberately omitted) and `set` (no `pp` scoremode value, deliberately absent because the server has no pp system).

### 3. Player & referee management

| Command | Syntax | Description |
|---|---|---|
| Add referee | `!mp addref <name>` | Adds a referee to the match — any existing referee can call this |
| Remove referee | `!mp removeref <name>` | Removes a referee — removing the last referee of a `!mp make` room will **auto-close it** |
| List referees | `!mp listrefs` | Lists the match's referees |
| Kick | `!mp kick <name>` | Removes a player from the room (does not block rejoin) |
| Ban | `!mp ban <name>` | Kicks + permanently blocks rejoin until `unban` |
| Unban | `!mp unban <name>` | Unblocks a previously banned player |
| Ban list | `!mp banlist` | Lists currently banned players for this match |

```
!mp addref RefereeName
→ BasilBot: "Added RefereeName to the match referees"
   (user not found online → "User not found: RefereeName")

!mp removeref RefereeName
→ BasilBot: "Removed RefereeName from the match referees"
   (removing the last referee of a !mp make room → "Removed RefereeName from the match referees. No referees
    remain — match closed")

!mp listrefs
→ BasilBot: "Match referees:"
→ BasilBot: "PlayerOne"
→ BasilBot: "RefereeName"
   (no referees yet → "No referees")

!mp kick TrollPlayer
→ BasilBot: "Kicked TrollPlayer from the match"

!mp ban CheaterPlayer
→ BasilBot: "Banned CheaterPlayer from the match"

!mp unban CheaterPlayer
→ BasilBot: "Unbanned CheaterPlayer from the match"

!mp banlist
→ BasilBot: "Match bans:"
→ BasilBot: "CheaterPlayer"
   (no bans yet → "No banned players")
```

**Kick/ban can target other referees** — only removes physical presence from the room; kicked/banned referees retain their `!mp` permissions and can still control the match remotely via `!mp in`. `ban` only applies to players physically present in the room when you issue the command (cannot look up offline players).

### 4. Match control

| Command | Syntax | Description |
|---|---|---|
| Start | `!mp start [seconds]` | Starts the match immediately, or countdown then auto-start if `seconds` is given |
| Timer | `!mp timer [seconds]` | Countdown like `start` but **does not** auto-start when time runs out (default 30s) |
| Abort timer | `!mp aborttimer` | Cancels a running countdown (both from `start` and `timer`) |
| Abort | `!mp abort` | Aborts the match in progress, resets ready/loaded, closes the current round |
| Close | `!mp close` | Closes the match immediately, kicks everyone, records end time |

```
!mp start
→ BasilBot: "Match started"

!mp start 30
→ BasilBot: "Match starts in 30 seconds"
   (then BasilBot auto-broadcasts to the match chat, not as a reply to you alone:
    "Queued the match to start in 30 seconds" → "Match starts in 10 seconds" → "Match starts in 5 seconds" →
    ... → "Good luck, have fun!")

!mp timer 60
→ BasilBot: "Countdown started: 60 seconds"
   (broadcast similar to start but ends with "Countdown finished" instead of auto-starting)

!mp aborttimer
→ BasilBot: "Countdown aborted"
   (no countdown running → "No countdown is running.")

!mp abort
→ BasilBot: "Aborted the match"
   (match not in progress → "Match is not in progress.")

!mp close
→ BasilBot: "Closed the match"
```
